# AGENTS.md — 高一数学暑假预习项目

本文件是面向 AI 协作代理的项目上下文入口。任何接手本项目的代理在无需人工解释的情况下,应能据此理解项目全貌、目录约定、生产规范与质量标准。

## 项目概述

服务对象为一名大学家教老师与一名初中基础较好的准高一学生。核心目标是用 20 天暑假完成人教 A 版 2019 版《必修第一册》的纯预习——不补初中,不延伸到必修第二册或选择性必修。每天授课 2 小时,共 40 小时。第一天为提纲挈领课,聚焦高中数学全貌、知识体系与学习方法,不涉及具体公式推导。

## 教材依据锁定

教材为人教 A 版 2019 版《必修第一册》(高一上学期用书),五章结构为不可更改的事实基础,任何内容生产必须以此为准:

- 第一章 集合与常用逻辑用语(1.1–1.5)
- 第二章 一元二次函数、方程和不等式(2.1–2.3)
- 第三章 函数的概念与性质(3.1–3.4)
- 第四章 指数函数与对数函数(4.1–4.5 + 数学建模)
- 第五章 三角函数(5.1–5.7)

官方电子课本入口为国家中小学智慧教育平台(basic.smartedu.cn)教材版块与人民教育出版社官网(pep.com.cn);第三方辅助源为电子课本网(dzkbw.com/books/rjb/shuxue/gzabx1)。交付物中给出平台入口而非深层直链,说明在平台内搜索"数学 必修第一册"定位。

## 课时框架

20 天 × 2 小时,Day 1 固定为提纲挈领课,章节课时分配为:

- Day 1:提纲挈领(全册导览)
- Day 2-4(3 天):第一章 集合与常用逻辑用语
- Day 5-7(3 天):第二章 一元二次函数、方程和不等式
- Day 8-11(4 天):第三章 函数的概念与性质
- Day 12-15(4 天):第四章 指数函数与对数函数
- Day 16-20(5 天):第五章 三角函数

合计 1+3+3+4+4+5=20 天。若某章进度滞后,压缩小结课而非核心概念课。任何课时调整需同步更新两份 HTML 报告。

## 学生画像与教学边界

学生初中数学基础较好,可直接进入高一预习,不安排初中内容补习。预习范围严格限定在必修第一册五章,不延伸到必修第二册或选择性必修。教学起点为初高中衔接处(如集合的直觉、二次函数的初中基础),而非初中知识系统复习。

## 写作风格约定

交付物正文遵循 doc-writing-guide 约定:

- 使用描述性、解释性语言,段落间有逻辑流动,而非命令式清单伪装的散文。
- 禁用"可操作/检查清单/定期复盘/x 天行动计划"等企业套话,禁用"进行/开展/实施+名词"的虚动词结构。
- 段落首句不重复标题已表达的内容,不用"下面我们来看"等元叙述,不用"这不是…而是…"的宣言式开场。
- 结论在其规范位置只陈述一次,其他章节引用时须提供新证据或新角度,而非换词重述。
- 技术术语随文给出简短解释,不堆砌生造抽象名词。
- 加粗仅用于专有名词、工具/命令名、关键警示,每节不超过 3 处;禁止加粗+斜体叠加;标题不加标点。

AGENTS.md 自身作为机器协作元文档,本质是规范性约定,采用"叙述说明 + 必要结构化列表"的混合形态——能用一句话说清的规则用叙述,涉及多要素的约定(如目录树、命名规范)用列表。

## HTML 生产规范

两份 HTML 报告使用 html-report 技能生产,遵循以下技术约定:

- 脚手架:用 `new-report.sh <report-name> <output-dir>` 创建目录结构,HTML 首行必须是 `<!-- Generated by Trae Work -->` 注释。
- 字体:canvas-fonts 中的字体均为拉丁字体,中文内容使用系统字体栈 `"PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif`;拉丁字符与数字可搭配 `Outfit` 或 `InstrumentSans`。
- 配色:所有颜色通过 CSS 变量定义在 `:root`,CSS 规则中只引用 `var(--*)`,不硬编码 hex。教育场景默认 Fresh Gradient 风格——低饱和度径向渐变底色(透明度 0.06-0.08)+ 半透明白色毛玻璃卡片(`rgba(255,255,255,0.7)` + `backdrop-filter: blur(10px)`),accent 色选沉稳靛蓝系(如 `#2C5F7C`)。
- ECharts:图表逻辑写入 `assets/charts.js`,用 IIFE 包裹,颜色通过 `getComputedStyle` 从 CSS 变量派生,不用彩虹色板或默认调色板;每张图表必须有标题(`<figcaption>` 或前导标题);`animation: false`;包含 resize 监听。`<script src="./_shared/js/echarts.min.js">` 必须在 `</body>` 前。
- Mermaid:`<pre class="mermaid">` 块写图源码,`mermaid.initialize({ startOnLoad: true, theme: 'neutral', securityLevel: 'loose' })`;`<script src="./_shared/js/mermaid.min.js">` 在 `</body>` 前。
- 禁止:CDN URL、绝对本地路径、base64 资源、内联 `<script>` 写图表逻辑。
- 关键词高亮:第一天报告采用 Method B(`<mark class="key">` + accent 色),正文加 `<mark class="key">` 高亮关键判断。
- 表格:用 `<div class="table-wrap">` 包裹,移动端可滚动;4 列以上表格用单列全宽布局,不放进多列网格。
- 移动端:max-width 容器居中,网格布局在小屏折叠为单列。

## 命名与版本规范

报告 slug 用英文小写连字符(如 `math-syllabus`、`day1-overview`),HTML 内中文标题作为展示名。文档更新在 AGENTS.md 末尾保留变更记录(日期 + 变更摘要)。

## 资源引用规范

HTML 中引用教材目录、官方平台信息时,用 `<sup><a href="#cite-N">[N]</a></sup>` 标注,footer `<div class="sources">` 内用 `<ol>` 给出完整 URL 的可点击链接。优先引用 smartedu.cn、pep.com.cn,第三方源(dzkbw.com)作辅助。

## 协作工作流

研究(验证教材与目录)→ 规划(确认天数与结构)→ 撰写(脚手架 → 内容 → 可视化)→ 校对(对照五章目录与写作约定)。每份交付物独立生产,互不阻塞。

## 质量校验要点

交付前核验:五章目录与官方一致、20 天天数合计为 20、Day 1 内容聚焦提纲挈领、HTML 在浏览器中图表正常渲染、无 CDN 与外链依赖、中文语言一致、正文无套话与元叙述、引用规范完整。
